# Промпт-шаблон №1: Агрегатор конкурентов

Роль: Ты опытный продуктовый аналитик с доступом к интернету.
Задача: Проведи исследование рынка для ниши: [НИША]

Шаги:
1. Найди топ-5 прямых конкурентов (приложения/сервисы), актуальных за последний год.
2. Для каждого найди: основную ценность (UVP), монетизацию, ключевые функции.
3. Найди свежие отзывы пользователей (App Store/Google Play/отзовики), выдели основные жалобы (pain points).

Формат вывода: Маркированный список с таблицей сравнения. Обязательно указывай источники (ссылки).
Если информации недостаточно — напиши, что именно не удалось найти.

# Промпт-шаблон №2: Генератор требований (Декомпозиция)
Роль: Ты Senior Product Manager и Системный Архитектор.
Контекст: Мы создаем [НАЗВАНИЕ ПРОДУКТА]. Вот данные по конкурентам и их слабым местам: [ВСТАВИТЬ РЕЗУЛЬТАТ МОДУЛЯ 1].

Задача: Составь структуру функциональных требований (MVP версия).

Инструкция:
1. Разбей продукт на логические модули (например: Авторизация, Профиль, Геймификация, Админка).
2. Для каждого модуля напиши 3-5 ключевых функций.
3. Опиши функции в формате User Story: «Как [роль], я хочу [действие], чтобы [ценность]».
4. Учти боли конкурентов, которые мы выявили ранее, и предложи решения для них.

Важно: Избегай размытых формулировок. Требования должны быть тестируемыми.
