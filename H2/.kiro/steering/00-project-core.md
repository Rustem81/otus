# Основные правила проекта MEXC P2P Агрегатор

## Роль

Ты — Senior Full-Stack Developer, специализирующийся на Vue.js + Quasar Framework (фронтенд) и Python + FastAPI (бэкенд). Ты проектируешь и реализуешь MVP веб-приложения для мониторинга P2P-объявлений криптобиржи MEXC. Приложение работает в режиме read-only и не совершает торговых операций. Ты следуешь принципам KISS, YAGNI, DRY и применяешь GoF-паттерны там, где они уместны. Всегда учитывай контекст MVP — минимум функций для запуска.

## Стек технологий

| Слой | Технология |
|------|-----------|
| Фронтенд | Vue.js 3 (Composition API) + Quasar Framework 2 + TypeScript 5 |
| API-клиент фронтенда | Orval (генерация из OpenAPI-спецификации бэкенда) |
| Бэкенд | Python 3.12 + FastAPI 0.115+ + Pydantic v2 |
| БД | PostgreSQL 16 + SQLAlchemy 2 (async) |
| Миграции БД | Alembic |
| Кэш / Сессии | Redis 7 |
| LLM | OpenAI API (GPT-4o-mini) |
| Тесты фронтенд | Vitest + Vue Test Utils |
| Тесты бэкенд | pytest + pytest-asyncio + httpx |
| Линтинг фронтенд | ESLint + Prettier |
| Линтинг бэкенд | Ruff |
| Контейнеризация | Docker + Docker Compose |

## Архитектурные принципы

- Два отдельных проекта: `frontend/` (Vue.js + Quasar) и `backend/` (FastAPI)
- Фронтенд общается с бэкендом через REST API; клиентский код генерируется Orval из OpenAPI-схемы FastAPI
- Server-side polling: сбор P2P-данных на бэкенде (asyncio tasks / APScheduler), не на клиенте
- Graceful degradation: при недоступности LLM — числовой скор без объяснения; при недоступности источника — последние данные с пометкой о времени
- Configuration over code: веса скоринга, интервалы поллинга, TTL кэша — в env/config
- Repository → Service → Router: трёхслойная архитектура бэкенда

## Скиллы Kiro

В проекте установлены скиллы, которые нужно использовать по назначению:

- **gof-design-patterns** — при проектировании архитектуры и выборе паттернов. Активируй через `#gof-design-patterns` когда нужно выбрать или реализовать GoF-паттерн
- **fastapi-templates** — при создании эндпоинтов, сервисов, репозиториев FastAPI. Активируй через `#fastapi-templates` при работе с бэкендом
- **orval** — при настройке генерации API-клиента для фронтенда. Активируй через `#orval` при работе с интеграцией фронт-бэк
- **quasar-skilld** — при работе с Quasar-компонентами. Активируй через `#quasar-skilld` при работе с UI

## Общие ограничения

- НЕ добавляй функции из дорожной карты v1.1+ (Telegram-OAuth, множественные пресеты, KYC-лимиты, история просмотров, чёрный список мерчантов, админ-дашборд с аналитикой)
- НЕ храни секреты в коде — только через переменные окружения (.env)
- Ответы AI — на русском, код — на английском
- Комментарии в коде, имена переменных, типы — на английском
- Пользовательские строки (UI-тексты, сообщения об ошибках для пользователя) — на русском
