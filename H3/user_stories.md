# User Story Mapping: MEXC P2P Агрегатор

## Пользовательские роли (Personas)

### Persona 1: Алексей — начинающий P2P-трейдер
- **Возраст:** 25 лет
- **Опыт:** 2 месяца на P2P
- **Потребность:** Понятный интерфейс, оценка рисков, подсказки
- **Страх:** Нарваться на мошенника, потерять деньги
- **Банки:** Сбербанк, Тинькофф

### Persona 2: Марина — активный P2P-трейдер
- **Возраст:** 32 года
- **Опыт:** 2 года на P2P, 50+ сделок в месяц
- **Потребность:** Скорость, точный расчёт доходности, фильтры
- **Страх:** Упустить выгодную сделку, переплатить комиссию
- **Банки:** Тинькофф, Райффайзен, СБП

### Persona 3: Дмитрий — администратор сервиса
- **Возраст:** 28 лет
- **Роль:** DevOps / техподдержка
- **Потребность:** Мониторинг интеграций, управление настройками
- **Страх:** Пропустить сбой источника данных

## User Stories

### Must Have (обязательно для MVP)

#### US-1: Регистрация пользователя
**Как** начинающий трейдер, **я хочу** зарегистрироваться по e-mail, **чтобы** иметь персональные настройки и сохранять фильтры.

**Критерии приёмки:**
- Given пользователь на странице регистрации When он вводит валидный e-mail и пароль (≥8 символов, буквы + цифры) Then создаётся аккаунт и пользователь перенаправляется на онбординг
- Given пользователь вводит уже зарегистрированный e-mail When он нажимает «Зарегистрироваться» Then отображается ошибка «E-mail уже используется»
- Given пользователь вводит пароль короче 8 символов When он нажимает «Зарегистрироваться» Then отображается ошибка валидации с требованиями к паролю

#### US-2: Онбординг-мастер
**Как** новый пользователь, **я хочу** при первом входе пройти мастер настройки (банки, суммы, риск-профиль), **чтобы** сразу получить персонализированный список объявлений.

**Критерии приёмки:**
- Given пользователь впервые входит в систему When загружается главная страница Then автоматически открывается онбординг-мастер из 4 шагов
- Given пользователь на шаге «Выбор банков» When он выбирает минимум 1 банк Then кнопка «Далее» становится активной
- Given пользователь на любом шаге онбординга When он нажимает «Пропустить» Then онбординг закрывается и применяются настройки по умолчанию

#### US-3: Просмотр таблицы P2P-объявлений
**Как** трейдер, **я хочу** видеть таблицу объявлений с ключевыми данными (мерчант, цена, спред, банки, риск-скор), **чтобы** быстро сравнивать предложения.

**Критерии приёмки:**
- Given пользователь на главном экране When данные загружены Then отображается таблица с колонками: мерчант, рейтинг, сделки, цена, спред, банки, лимиты, объём, риск-скор
- Given таблица отображена When проходит 15 секунд Then данные автоматически обновляются без перезагрузки страницы
- Given в системе нет объявлений по выбранным фильтрам When пользователь смотрит таблицу Then отображается сообщение «Нет объявлений по заданным критериям» с предложением изменить фильтры

#### US-4: Фильтрация объявлений
**Как** активный трейдер, **я хочу** фильтровать объявления по банкам, рейтингу, сумме и способу оплаты, **чтобы** видеть только релевантные предложения.

**Критерии приёмки:**
- Given пользователь на главном экране When он выбирает банк «Тинькофф» в фильтре Then таблица показывает только объявления с поддержкой Тинькофф
- Given пользователь задаёт минимальный рейтинг 4.5 When фильтр применяется Then объявления с рейтингом ниже 4.5 скрываются
- Given пользователь задаёт диапазон суммы 10 000–50 000 RUB When фильтр применяется Then отображаются только объявления с лимитами, пересекающимися с этим диапазоном

#### US-5: Просмотр риск-скора и LLM-объяснения
**Как** трейдер, **я хочу** видеть риск-скор каждого объявления и получать AI-объяснение по клику, **чтобы** понимать надёжность мерчанта.

**Критерии приёмки:**
- Given объявление в таблице When пользователь смотрит колонку «Риск» Then отображается числовой скор (1–10) с цветовой индикацией (зелёный/жёлтый/красный)
- Given пользователь кликает на иконку рядом с риск-скором When LLM доступен Then появляется всплывающее окно с текстовым объяснением (до 300 символов)
- Given пользователь кликает на иконку рядом с риск-скором When LLM недоступен Then отображается сообщение «AI-объяснение временно недоступно» и числовой скор остаётся видимым

#### US-6: Расчёт чистой доходности
**Как** трейдер, **я хочу** видеть чистую доходность с учётом комиссий моего банка, **чтобы** не переоценивать выгодность сделки.

**Критерии приёмки:**
- Given пользователь указал комиссию банка в профиле When он смотрит таблицу Then колонка «Чистая доходность» показывает процент с учётом комиссии
- Given пользователь наводит курсор на значение чистой доходности When появляется tooltip Then в нём отображается детализация: цена, спред, комиссия, итоговый PnL
- Given пользователь не указал комиссию банка When он смотрит таблицу Then чистая доходность рассчитывается с нулевой комиссией и отображается пометка «без учёта комиссий»

#### US-7: Сохранение набора фильтров
**Как** активный трейдер, **я хочу** сохранить текущий набор фильтров как «сценарий по умолчанию», **чтобы** при следующем входе не настраивать всё заново.

**Критерии приёмки:**
- Given пользователь настроил фильтры When он нажимает «Сохранить как по умолчанию» Then текущие фильтры сохраняются в профиле
- Given пользователь имеет сохранённый сценарий When он входит в систему Then фильтры автоматически применяются

### Should Have (желательно)

#### US-8: Карточка объявления
**Как** трейдер, **я хочу** по клику на объявление видеть полную информацию в модальном окне, **чтобы** принять решение без переключения экранов.

**Критерии приёмки:**
- Given пользователь кликает на строку в таблице When данные загружены Then открывается модальное окно с полными данными: все метрики мерчанта, банки, лимиты, LLM-резюме
- Given модальное окно открыто When пользователь нажимает «Открыть в MEXC» Then открывается новая вкладка с соответствующим объявлением на MEXC

#### US-9: Сортировка таблицы
**Как** трейдер, **я хочу** сортировать объявления по доходности, рейтингу, объёму или риск-скору, **чтобы** выбирать оптимальный критерий.

**Критерии приёмки:**
- Given таблица с объявлениями When пользователь кликает на заголовок колонки «Чистая доходность» Then объявления сортируются по убыванию доходности
- Given таблица отсортирована по доходности When пользователь кликает на заголовок повторно Then сортировка меняется на возрастающую

#### US-10: Профиль трейдера — банки и суммы
**Как** трейдер, **я хочу** в настройках профиля управлять списком банков, диапазоном сумм и валютными парами, **чтобы** обновлять свои предпочтения.

**Критерии приёмки:**
- Given пользователь в разделе «Профиль» When он добавляет банк «Райффайзен» Then банк появляется в списке и фильтры на главном экране обновляются
- Given пользователь меняет диапазон сумм When он сохраняет профиль Then новый диапазон применяется к фильтрации объявлений

#### US-11: Чёрный список мерчантов
**Как** трейдер, **я хочу** добавлять мерчантов в чёрный список, **чтобы** их объявления больше не отображались.

**Критерии приёмки:**
- Given пользователь в карточке объявления When он нажимает «Заблокировать мерчанта» Then объявления этого мерчанта скрываются из таблицы
- Given мерчант в чёрном списке When пользователь открывает настройки чёрного списка Then он может удалить мерчанта из списка

### Nice to Have (бонус)

#### US-12: История просмотренных объявлений
**Как** трейдер, **я хочу** видеть список последних просмотренных объявлений, **чтобы** вернуться к ним позже.

**Критерии приёмки:**
- Given пользователь открывал карточку объявления When он переходит в раздел «История» Then отображается список последних 50 просмотренных объявлений с датой и основными параметрами

#### US-13: Админка — статус интеграций
**Как** администратор, **я хочу** видеть статус источников данных (OK/Degraded/Down) и количество ошибок, **чтобы** оперативно реагировать на сбои.

**Критерии приёмки:**
- Given администратор в админ-панели When он открывает дашборд Then отображаются статусы всех источников и счётчики ошибок за 24 часа

#### US-14: Админка — управление валютными парами
**Как** администратор, **я хочу** включать/выключать валютные пары, **чтобы** оперативно реагировать на изменения рынка.

**Критерии приёмки:**
- Given администратор в разделе «Валютные пары» When он выключает пару RUB/USDT Then система прекращает сбор данных по этой паре и объявления скрываются из UI
