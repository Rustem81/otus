# Описание проекта: MEXC P2P Агрегатор

## Продуктовая идея

**MEXC P2P Агрегатор** — веб-приложение для мониторинга, анализа и ранжирования P2P-объявлений криптобиржи MEXC. Приложение работает в режиме read-only: не совершает торговых операций от имени пользователя и не требует API-ключей MEXC.

## Проблема

P2P-трейдеры на MEXC сталкиваются с рядом проблем:

1. **Информационная перегрузка.** Десятки объявлений с разными ценами, лимитами, способами оплаты — сложно быстро найти оптимальное предложение.
2. **Отсутствие оценки рисков.** Биржа показывает базовые метрики мерчанта (рейтинг, число сделок), но не даёт интегральной оценки надёжности.
3. **Скрытые издержки.** Комиссии банков и платёжных систем не учитываются при сравнении цен — трейдер переоценивает доходность.
4. **Ручная рутина.** Каждый раз нужно заново настраивать фильтры, проверять мерчантов, считать чистую доходность.

## Целевая аудитория

| Сегмент | Описание | Потребность |
|---------|----------|-------------|
| Начинающие P2P-трейдеры | Пользователи, которые только начинают торговать на P2P-площадках | Простой интерфейс, понятная оценка рисков, подсказки |
| Активные P2P-трейдеры (РФ) | Опытные трейдеры, работающие с RUB ⇄ USDT через российские банки | Быстрая фильтрация по банкам, расчёт чистой доходности с учётом комиссий |
| Администраторы сервиса | Технические специалисты, управляющие платформой | Мониторинг интеграций, управление валютными парами, контроль ошибок |

## Ключевые функции (MVP)

### Функция 1: Агрегация и фильтрация P2P-объявлений

Автоматический сбор P2P-объявлений MEXC с интервалом обновления ≤15 секунд. Таблица с колонками: мерчант, рейтинг, число сделок, цена, спред, банки, лимиты, объём, риск-скор. Фильтрация по банкам, рейтингу, сумме, способу оплаты. Сортировка по доходности, рейтингу, риск-скору. Сохранение набора фильтров как «сценарий по умолчанию».

### Функция 2: Риск-скоринг с LLM-объяснениями

Числовой риск-скор от 1 до 10 для каждого объявления на основе метрик мерчанта (рейтинг, число сделок, процент успешных, скорость закрытия). Маппинг в категории: низкий (1–3), средний (4–7), высокий (8–10). LLM-объяснение по клику (кэшируется 10 минут). Graceful degradation: при недоступности LLM показывается числовой скор без объяснения.

### Функция 3: Расчёт чистой доходности

Учёт комиссий банка/платёжного метода (фикс и/или процент) при расчёте доходности. Отображение «чистой доходности» в таблице для каждого объявления. Всплывающая подсказка с детализацией: цена, спред, комиссия, итоговый PnL.

### Функция 4 (дополнительная): Профиль трейдера

Выбор банков и способов оплаты, диапазонов сумм, валютных пар (RUB ⇄ USDT). Выбор риск-профиля (низкий/средний/высокий). Онбординг-мастер при первом входе. Указание KYC-уровня для предупреждений о лимитах.

### Функция 5 (дополнительная): Карточка объявления

Модальное окно с полными данными о мерчанте и сделке. Кнопка «Открыть в MEXC» для перехода к исполнению. Краткое LLM-резюме «подходит/не подходит» с учётом профиля пользователя.

## Ценностное предложение
